## Vector DB Use Case
In traditional databases like SQL, search is limited because the computer only looks for exact matches. This means that words like "Termination", "termination," and "terminations" are seen as completely different, even though they have the same meaning. While humans understand the meaning of a word, a computer usually only sees a string of characters. In a legal setting, a keyword search would fail if one contract uses the word "Termination" while another uses the word "Cancellation." These relevant documents would be missed.

This is where a vector database is used to solve the problem. Instead of storing text as plain words, the information is turned into a series of numbers called vectors. These vectors are mapped out in a mathematical "space" where their positions are based on their actual meaning. When these vectors are plotted, points that are placed closer together are considered to have similar meanings. This process is known as a semantic search.

When a lawyer asks a question in plain English, the query is also turned into a vector. A similarity search is then performed to find the sections of the contract that are mathematically closest to the question. Even if different vocabulary is used in the document, the correct information is found because the system is matching the intent rather than just the letters. By using this method, 500-page contracts can be searched in seconds, ensuring that no important legal clauses are missed.
